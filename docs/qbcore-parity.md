# QBCore Parity Track

Status: the active QBCore parity backlog is currently cleared.

This track exists so QBCore compatibility work has one clear backlog. The active backlog is listed first and must contain only work that is not done yet. Verified coverage is kept below as evidence, not as pending work.

## Active Backlog

No active QBCore parity backlog is currently listed. Add newly discovered compatibility gaps here only after they are reproduced with a failing parity or migration test.

## Grounding

Current QBCore compatibility needs to cover the public surfaces real FiveM resources expect:

- Core object access through `exports['qb-core']:GetCoreObject()` and, in newer QBCore versions, direct player exports such as `exports['qb-core']:GetPlayer(source)` and `exports['qb-core']:GetPlayerData()`.
- Server player lookup helpers such as `QBCore.Functions.GetPlayer`, `GetPlayers`, `GetQBPlayers`, `GetPlayersByJob`, `GetPlayerByCitizenId`, `GetPlayerByPhone`, `GetPlayerByAccount`, `GetPlayerByCharInfo`, duty counts, callbacks, usable items, permissions, commands, and vehicle helpers.
- Client helpers such as `GetPlayerData`, `Notify`, `TriggerCallback`, `GetCoords`, proximity helpers, vehicle helpers, and `HasItem`.
- Player object methods for character data, jobs, gangs, money, metadata, inventory checks, save/logout, and custom fields or methods.

Sources used for this parity map:

- [QBCore server function reference](https://docs.qbcore.org/qbcore-documentation/qb-core/server-function-reference)
- [QBCore client function reference](https://docs.qbcore.org/qbcore-documentation/qb-core/client-function-reference)
- [QBCore player data reference](https://docs.qbcore.org/qbcore-documentation/qb-core/player-data)
- [qb-core server functions source](https://github.com/qbcore-framework/qb-core/blob/main/server/functions.lua)

## Non-Goals For First Parity Pass

- Do not clone every QBCore implementation detail internally.
- Do not reintroduce loose MySQL scripts as source of truth.
- Do not let compatibility exports mutate authoritative state outside reducers/action handlers.
- Do not treat `provide 'qb-core'` as complete parity by itself; it only satisfies dependency resolution.
